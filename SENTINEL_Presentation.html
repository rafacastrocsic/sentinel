Scientific Context

🔬 CMOS Degradation Phenomena

  • BTI (Bias Temperature Instability)
  • HCD (Hot-Carrier Degradation)
  • TDDB (Time-Dependent Dielectric Breakdown)
  • RTN (Random Telegraph Noise)
  • Electromigration

⚠️ Current Technology Limitations

  • ✕ Software-level monitoring: No access to transistor-level degradation
  • ✕ System-level health checks: Reactive, not predictive
  • ✕ External testing: Expensive, offline, cannot monitor in-field
  • ✕ No dual-purpose exploitation: Security and reliability treated separately

The Solution: Project SENTINEL

🎯 Project Objective

SENTINEL will exploit CMOS variability phenomena (BTI, HCD, TDDB, RTN) to simultaneously enhance chip security and reliability through a unified hardware-level approach, delivering an integrated FPGA/SoC demonstrator validated in defence/critical infrastructure scenarios.

⚡ Problem to Solve

AI accelerator aging threatens data-centre reliability. Semiconductor counterfeiting costs >$100B/year. No existing platform exploits on-chip degradation for both security and health monitoring simultaneously.

📅 Gantt Chart

SENTINEL Project Timeline (42 months)

🔗 Critical Path

The sequence of interdependent activities that determines the minimum project duration. Any delay on the critical path delays the entire project.

1. Secure Chip Primitives (WP2)

M1 → M30

Blocks WP5 integration. Buffer: 0 months (critical). PUF + PQC deliverables must be ready for T5.1.

2. On-Chip Monitoring & ML (WP3)

M4 → M34

Parallel to WP2. Blocks WP5 via sensor arrays and ML inference engine. MS4 Go/No-Go at M22.

3. Integration & Validation (WP5)

M18 → M40

The critical integration gate. Depends on WP2 and WP3. MS7 Go/No-Go at M30. Buffer: 10 months for validation.

4. Security-by-Design (WP4)

M6 → M36

Feeds into WP5 with more schedule margin. Not on critical path but essential for certification compliance.

💡 Strategic Buffers

WP2 → WP5 handoff: WP2 ends at M30, WP5 integration starts at M18. Overlap of 12 months allows incremental delivery of PUF/PQC blocks to WP5 as they mature, rather than a single handoff.

WP3 → WP5 handoff: WP3 delivers ML inference engine at M34, while WP5 integration starts at M18. Sensor array spec (D3.1, M10) enables early integration work.

Fabrication contingency: IMEC pilot line as backup to commercial MPW runs. Parallel FPGA emulation path ensures integration work is never blocked by silicon delays.

Validation window: MS7 (demonstrator operational) at M30 leaves 10 months for pilot validation (T5.4), non-destructive analysis (T5.2), and supply chain demo (T5.3) before project end at M42.

⏱️ Total Project Duration: 42 months with ~10% implicit contingency distributed across non-critical WPs (WP4, WP6, WP7 all run to M42, providing flexibility).

⚠️ Risk Response Strategy

Avoidance: All fabrication risks mitigated via redundant pathways (IMEC pilot + commercial MPW + FPGA emulation).

Monitoring: Monthly risk reviews by Scientific Committee; quarterly reports to EC via WP1.
